# Computer-Using Agent vs Agent-Native OS

## Computer-Using Agent

Computer-use systems teach agents to operate human interfaces:

- screen reading
- button clicking
- form filling
- browser automation
- terminal execution
- API/tool calls
- GUI workflows

This is useful. It gives agents hands and eyes inside human computing environments.

## Agent-Native OS

Agent-Native OS gives agents a native working environment:

- context budgeting
- source hierarchy
- task isolation
- role runtime
- output contracts
- audit gates
- handoff protocols
- memory and archive
- recovery points
- Skill App runtime

## Relationship

Computer-use is a driver.

Agent OS is the governance layer.

```txt
Agent-Native OS
  ├─ Context Kernel
  ├─ Skill App Runtime
  ├─ Source File System
  ├─ Role Runtime
  └─ Tool Drivers
      ├─ Browser
      ├─ Terminal
      ├─ API
      ├─ MCP
      └─ GUI Control
```

The question is not "Can the agent click?"

The question is:

- why is it clicking?
- under which task?
- with which permission?
- using which sources?
- producing which output?
- audited by whom?
- recoverable how?
