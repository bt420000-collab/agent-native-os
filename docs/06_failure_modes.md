# Agent OS Failure Modes

## 1. Context Overflow

Too much context enters the session. The agent loses priority and starts blending sources.

## 2. Role Bleeding

A role exceeds its function. A writer audits, an auditor rewrites, an archivist invents requirements.

## 3. Source Pollution

Old drafts, deprecated decisions, or brainstorm material contaminate the current source of truth.

## 4. Handoff Amnesia

A sub-agent completes work but fails to produce a usable continuation state.

## 5. Output Drift

The output gradually deviates from the required format, purpose, or style.

## 6. Unauthorized Mutation

An agent modifies core sources or mainline files without permission.

## 7. Review Collapse

Important work enters the mainline without audit.

## 8. Archive Haunting

Cold or deprecated materials keep returning to influence current work.

## 9. User Override Loss

The user's latest instruction is not promoted into the proper authority level.

## 10. Long-Run Degradation

The project becomes less coherent over time as context, roles, and source authority decay.

## Why failure modes matter

A system becomes real when it names its failure modes.

Agent OS is designed around preventing and recovering from these failures.
