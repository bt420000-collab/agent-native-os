# Why Agent OS?

AI agents are getting more tools: browsers, terminals, APIs, MCP servers, databases, file editors, and GUI controllers.

That solves the problem of action.

It does not solve the problem of governance.

Long-running agent work fails in predictable ways:

- the agent reads too much and loses priority
- a worker changes core sources without permission
- a reviewer starts rewriting instead of auditing
- a deprecated draft pollutes current work
- task output is conversational instead of reusable
- the handoff is vague
- the next session cannot continue safely
- user overrides are not promoted into source law
- no recovery point exists after a bad mutation

Agent OS exists because intelligent action without operating discipline becomes chaos.

The purpose is not to make agents autonomous in the abstract.

The purpose is to make agent work governable, inspectable, auditable, recoverable, and composable.
