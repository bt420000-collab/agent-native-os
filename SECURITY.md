# Security Policy

Agent-Native OS is currently a specification-first project. Security feedback is especially welcome around permissions, source authority, audit gates, handoff protocols, and recovery behavior.

## Reporting a Vulnerability

Please do not disclose exploitable issues publicly before maintainers have had a chance to review them.

Use one of these channels:

- Open a private GitHub security advisory if the repository has advisories enabled.
- Email the maintainer privately at bt420@126.com.
- Contact the maintainer on WeChat: iMelodyJust.
- If neither option is available yet, open a minimal public issue that says you have a security report to share, without including exploit details.

## What to Include

- Affected file, template, schema, or workflow.
- Why the behavior could cause unauthorized access, source pollution, hidden mutation, output drift, or recovery failure.
- A minimal reproduction or example, if possible.
- Suggested mitigation, if you have one.

## Supported Versions

Only the latest public version is actively maintained unless a release branch states otherwise.
