# Agent-Native OS Manifesto

## 1. Agents do not need more buttons first. They need a workplace.

The industry is teaching agents to use human computers: screens, buttons, menus, browsers, terminals, and legacy workflows.

That is useful.

But it does not answer the deeper question:

> What kind of operating environment does an agent need for its own work?

A human OS was built for eyes, hands, cursors, windows, and applications.

An Agent OS must be built for context, sources, roles, tasks, contracts, audits, handoffs, memory, and recovery.

## 2. Context is the first-class resource

The primary resource of an agent is not CPU, memory, or screen space.

It is context.

Agents fail when their context is bloated, polluted, stale, contradictory, unauthorized, or ungoverned.

A context-native OS should decide:

- what an agent can know
- what an agent can change
- what an agent must produce
- what an agent must not inherit
- how an agent hands off state
- how work survives beyond one session

## 3. Tools need governance

Browsers, APIs, terminals, MCP servers, databases, file editors, and GUI controllers are drivers.

They are not the OS.

The OS layer must decide:

- who can call a tool
- under what task
- with what permissions
- against which sources
- producing which outputs
- through which audit gate
- with what recovery point

More tools without governance create faster chaos.

## 4. Skill Apps are the applications of Agent OS

In a human OS, users install applications.

In an Agent OS, developers install Skill Apps.

A Skill App is not just a prompt. It is a governed package with:

- manifest
- instructions
- input contract
- output contract
- permissions
- source policy
- audit requirements
- examples
- tests
- changelog

A Skill App should be installable, inspectable, composable, auditable, and removable.

## 5. Agent work must be governable

Reliable long-running agent work requires:

- task isolation
- source hierarchy
- role runtime
- output contracts
- audit gates
- handoff protocols
- cold archives
- recovery points

The point is not full autonomy.

The point is governable autonomy.

## 6. The goal

Agent-Native OS aims to become an open architecture for context-native agent workspaces.

It is the Windows layer for agents:

> not a model, not a tool, not a workflow, but the system in which models, tools, skills, and workflows become reliable work.
